---Estruturação da equipe---

O repositorio foi configurado para não permitir commits diretamente na master, so sendo possivel commitar fazendo uso de uma pullRequest que no minumo tem que ser aprovada por pelo menos uma pessoa
Cada integrante realizava a programação em uma branch separada dentro de sua maquina, após terminar o codigo esta pessoa realizava um push e fazia o pedido de revisão. 
Por fim, se aprovada as mudanças, era realizado o merge com a branch master

- Lider Gustavo
    Teve como responsabilidade a criação do repositorio, esqueleto inicial e resposavel pela revisão dos codigos por meio dos PullRequest.
- Outros integrantes
    Cada um programou partes esspecificas do sistema assim como corrigiu erros ou pedidos de mudança de codigo após a revisão pelo lider de equipe

---conflitos e como foram resolvidos---

Ocorreu apenas um conflito na etapa final do desenvolvimento do sistema, porém foi facilmente resolvido utilizando as ferramentas fornecidas pelo visual studio code

---dificuldades encontradas---

Além do processo de abtuação com o funcionamento do git integrado com a plataforma github, não houveram maiores dificuldades
